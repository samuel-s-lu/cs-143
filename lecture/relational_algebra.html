Intro

  • recall that a relation is a set of tuples
    • each tuple contains one or more named attributes
  • order of tuples in a relation doesn't matter (set)
  • order of attributes in each tuple must be consistent

Common Operations on Data

  • Selection σ\sigmaσ
    • retrives a subset of tuples from a single relation that satisfies a particular constraint and returns a new relation which is a subset of the original
    • σψ(R)={t∈R:ψ(t)}\sigma_\psi(R) = \cbrack{t\in R:\psi(t)}σψ​(R)={t∈R:ψ(t)}
      • ψ\psiψ is a predicate that compares two attributes or an attribute and a value using a binary operator
        • we can build complex predicates using conjunction ∧\land∧ and disjunction ∨\lor∨
    • SQL WHERE acts as a filter
      • we can write it in relational algebra as
        σl>d∧v>1M∧c=24(R)\sigma_{l>d\land v>1M\land c=24}(R)σl>d∧v>1M∧c=24​(R)
      • note: σ≠SQL SELECT\sigma\neq SQL\ SELECTσ=SQL SELECT
      • similar to SELECT * FROM RRR WHERE ψ\psiψ
  • Projection Π\PiΠ
    • extracts subsets of attributes from relations and removes duplicates
    • Πa1,…,an(R)={t[a1,…,an]:t∈R}\Pi_{a_1,\dots,a_n}(R)=\cbrack{t[a_1,\dots,a_n]:t\in R}Πa1​,…,an​​(R)={t[a1​,…,an​]:t∈R}
    • Π=SQL SELECT DISTINCT\Pi=SQL\ SELECT\ DISTINCTΠ=SQL SELECT DISTINCT
    • can be generalized to "create" new attributes using the →\to→ notation
      • apply arbitrary expressions to attributes
        • Πlikes/(likes+dislikes)\Pi_{likes/(likes+dislikes)}Πlikes/(likes+dislikes)​
      • apply arbitrary expressions to create new columns
        • Πlikes+dislikes→interactions\Pi_{likes+dislikes\to interactions}Πlikes+dislikes→interactions​
      • rename attributes
        • Πthumbs_up→likes\Pi_{thumbs\_up\to likes}Πthumbs_up→likes​
      • generalized Π≈\Pi\approxΠ≈ SQL AS
    • if we want all attributes, just use σ\sigmaσ and not use Π\PiΠ
    • ex.
      • write a relational algebra to extract the title and channel for all videos that have more than 1 million views and more likes than dislikes
      • Πtitle,channel(σv>1M∧l>d(R))\Pi_{title, channel}(\sigma_{v>1M\land l>d}(R))Πtitle,channel​(σv>1M∧l>d​(R))
      • the SQL equivalent is SELECT DISTINCT
    • there can be more than one correct way to chain together operations
  • Cartesian Product ×\times×
    • forming all pairs of possible ordered tuples between two relation
    • R×S={r∪s:r∈R∧s∈S}R\times S=\cbrack{r\cup s:r\in R\land s\in S}R×S={r∪s:r∈R∧s∈S}
    • in practice you would rarely use the Cartesian product
      • very slow and memory intensive
      • we mostly use it to motivate the join
  • Join
    • we perform joins to merge tuples from two different relations based on some contextually related attribute(s) in both relations
    • results in another relation where each tuple contains data from the tuples of both relations that were joined
    • join key - the sets of related attributes that are used to join the two relations
      • has nothing to do with uniqueness
      • FK can be JK but FK ≠\neq= JK
    • every join is a natural join or a theta join
    • Natural Join ⋈\bowtie⋈
      • the RDBMS automatically picks the attribute(s) kkk to join on - the attribute names common to both relations
        • RDBMS identifies join key as any attribute that is present in both relations and has the same name
        • we get one copy of the join key in the output
        • we only discuss the case where the output only includes tuples that have a match in both relations (inner)

      • R⋈S=ΠR∪S(σR.k=S.k(R×S))={r∪s:r∈R∧s∈S∧r[k]=s[k]}\aligned{R\bowtie S &= \Pi_{R\cup S}(\sigma_{R.k=S.k}(R\times S)) \\ &= \cbrack{r\cup s:r\in R\land s\in S\land r[k]=s[k]} }R⋈S​=ΠR∪S​(σR.k=S.k​(R×S))={r∪s:r∈R∧s∈S∧r[k]=s[k]}​

      • Edge Cases:
        • joining two relations R,SR,SR,S that have no common attributes
          • k=∅  ⟹  r[k]=s[k]=∅k=\empty\implies r[k]=s[k]=\emptyk=∅⟹r[k]=s[k]=∅ so the predicate is always true
          • thus R⋈S=R×SR\bowtie S=R\times SR⋈S=R×S
        • joining two relations R,SR,SR,S that have common attributes but no mataches on those attributes
          • r[k]=s[k]r[k]=s[k]r[k]=s[k] is never true   ⟹  R⋈S=∅\implies R\bowtie S=\empty⟹R⋈S=∅
      • tuples are combined using equality of values in the common attributes
        • thus natural join AKA equijoin
    • Theta Join ⋈θ\bowtie_\theta⋈θ​
      • enforces an additional constraint θ\thetaθ on the natural join
        • we get two copies of the join key (even if names are different)
        • R⋈θS=σθ(R×S)={r∪s:r∈R∧s∈S∧θ(r∪s)}R\bowtie_\theta S=\sigma_\theta (R\times S)=\cbrack{r\cup s:r\in R\land s\in S\land \theta(r\cup s)}R⋈θ​S=σθ​(R×S)={r∪s:r∈R∧s∈S∧θ(r∪s)}
      • alias - we must use modified names for attribute names that are common to both relations
        • prefix attribute name with the relation name
      • theta joins are much more general and powerful compared to natural joins
      • if θ\thetaθ consists entirely of equality constraints, it is called an equijoin
        • otherwise it's a non-equijoin
  • Aggregation γ\gammaγ or G\mathcal{G}G
    • uses an aggregation function to a group of tuples in a relation to summarize them
    • common aggregation functions:
      • SUM
      • AVG
      • MIN
      • MAX
      • DISTINCT-COUNT
    • we ignore NULL values
    • Notation:
      • attributes to group byγfunctions to apply(Relation)_\text{attributes to group by}\gamma_\text{functions to apply}(Relation)attributes to group by​γfunctions to apply​(Relation)
    • examples:
      • average GPA of each major:
        • majorγAVG(gpa)→major_averages(students)_{major} \gamma_{AVG(gpa)\to major\_averages}(students)major​γAVG(gpa)→major_averages​(students)
      • global average GPA:
        • γAVG(gpa)→bruin_average(students)\gamma_{AVG(gpa)\to bruin\_average}(students)γAVG(gpa)→bruin_average​(students)
      • average GPA and average sleep of each major
        • majorγAVG(gpa),AVG(sleep)(students)_{major} \gamma_{AVG(gpa),AVG(sleep)}(students)major​γAVG(gpa),AVG(sleep)​(students)
  • Rename ρ\rhoρ
    • used to rename relations, attributes, or relational algebra subexpressions (of a bigger complex expression)
    • Notation:
      • ρS(R)\rho_S(R)ρS​(R) renames relation (or expression) RRR to SSS
        • ρnew(old)\rho_{new}(old)ρnew​(old)
      • ρa/b(R)\rho_{a/b}(R)ρa/b​(R) renames attribute bbb in relation RRR, to aaa
        • ρnew/old(R)\rho_{new/old}(R)ρnew/old​(R)
      • special case: in the Cartesian product, R×RR\times RR×R is not valid - we must alias one side as ρL(R)×R\rho_L(R)\times RρL​(R)×R
    • recall that the Projection operater Π\PiΠ can also be used to rename attributes
      • Πold→new\Pi_{old\to new}Πold→new​
    • Remark: ρ\rhoρ does not change the underlying data (just outputs a temp view)
  • Set Theory Operations
    • when performing a set theory operation in relational algebra, the 2 sets must be compatible, i.e. they both must have the same
      • number of attributes
      • attribute names
      • domain for each attribute
    • Set Union ∪\cup∪
      • R∪SR\cup SR∪S contains all tuples that are in RRR, in SSS, and in both RRR and SSS (discarding duplicates)
        • R∪S={r1,…,r∣R∣,s1,…,s∣S∣}R\cup S = \cbrack{r_1,\dots,r_{|R|},s_1, \dots, s_{|S|}}R∪S={r1​,…,r∣R∣​,s1​,…,s∣S∣​}
      • example:
        • imagine two relations R,SR,SR,S stored on different machines
          • have to send tuples to computation node
        • write an expression that yields the name of all students that have a GPA ≥3.0\geq 3.0≥3.0
        • 3 ways to write this
          1. Πname(σgpa≥3.0(R∪S))\Pi_{name}(\sigma_{gpa\geq 3.0}(R\cup S))Πname​(σgpa≥3.0​(R∪S))
          1. Πname(σgpa≥3.0(R)∪σgpa≥3.0(S))\Pi_{name}(\sigma_{gpa\geq 3.0}(R)\cup \sigma_{gpa\geq 3.0}(S))Πname​(σgpa≥3.0​(R)∪σgpa≥3.0​(S))
          1. Πname(σgpa≥3.0(R))∪Πname(σgpa≥3.0(S))\Pi_{name}(\sigma_{gpa\geq 3.0}(R))\cup \Pi_{name}(\sigma_{gpa\geq 3.0}(S))Πname​(σgpa≥3.0​(R))∪Πname​(σgpa≥3.0​(S))
        • the 3 ways progressively use less and less network I/O
      • example:
        • we can use ρ\rhoρ to make it work
        • ρgpa/ugrad_gpa(R)∪ρgpa/grad_gpa(S)\rho_{gpa/ugrad\_ gpa}(R)\cup \rho_{gpa/grad\_ gpa}(S)ρgpa/ugrad_gpa​(R)∪ρgpa/grad_gpa​(S)
    • Set Intersection ∩\cap∩
      • R∩SR\cap SR∩S retrives all tuples that are in both RRR and SSS
        • R∩S={t:t∈R∧t∈S}R\cap S=\cbrack{t:t\in R\land t\in S}R∩S={t:t∈R∧t∈S}
    • Set Difference −-−
      • R−SR-SR−S retreives every tuple in RRR that are not in SSS
        • R−S={t∈R:t∉S}R-S=\cbrack{t\in R:t\not\in S}R−S={t∈R:t∈S}
      • similar to SQL EXCEPT
